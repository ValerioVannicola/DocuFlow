# Results And Data Models

This file documents the public data models users interact with after ingestion, parsing,
extraction, validation, review, correction, search, and storage.

The main contract is `ExtractionResult`. It is the final object users get back from
high-level APIs, and it carries both the extracted values and the metadata needed to audit
how those values were produced.

Most models are Pydantic models and can be serialized with:

```python
obj.model_dump()
obj.model_dump_json(indent=2)
```

## `ExtractionResult`

Import:

```python
from docuflow.extraction.models import ExtractionResult
```

Returned by:

- `extract()`
- `DocumentPipeline.run()` / `run_sync()`
- `run_workflow()`
- Batch/comparison/routing workflows inside their report objects.

Fields:

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `document_id` | `str` | Required | Document UUID/id from ingestion. |
| `schema_name` | `str` | Required | Name of the schema class used for extraction. |
| `data` | `dict` | `{}` | Plain extracted values keyed by field name. |
| `fields` | `dict[str, ExtractedField]` | `{}` | Full field objects with value, trust gate, evidence, OCR/consensus data, and validation. |
| `confidence` | `float` | `0.0` | Overall extraction confidence/trust score. |
| `confidence_score` | `float \| None` | `None` | OCR-based confidence score for the final result. Prefers `result.ocr.score`, then falls back to the mean of available field OCR scores. |
| `consensus_score` | `float \| None` | `None` | Multi-instance consensus score for the final result. Average of available field agreement ratios. |
| `ocr` | `OCRDocumentConfidence \| None` | `None` | Document-level OCR confidence when OCR ran. |
| `usage` | `TokenUsage \| None` | `None` | Aggregated LLM token usage and cost. |
| `escalated` | `bool` | `False` | True when auto mode escalated to vision/hybrid. |
| `escalation_reason` | `str` | `""` | Human-readable auto escalation reason. |
| `needs_review` | `bool` | `False` | True when review rules found issues. |
| `review_status` | `str` | `"pending"` | `"pending"`, `"approved"`, or `"rejected"`. |
| `reviewed_by` | `str` | `""` | Reviewer identity. |
| `reviewed_at` | `datetime \| None` | `None` | Review timestamp. |
| `rejection_reason` | `str` | `""` | Reason supplied when rejected. |
| `review_reasons` | `list[str]` | `[]` | Reasons generated by review rules/reviewers. |
| `review_verdicts` | `list[ReviewVerdict]` | `[]` | LLM reviewer outputs. |
| `corrections` | `list[FieldCorrection]` | `[]` | Human correction history. |
| `validation_errors` | `list[dict]` | `[]` | Validation errors serialized from `ValidationError`. |
| `trace_id` | `str` | `""` | Trace identifier. |
| `model_name` | `str` | `""` | Model used for extraction. |
| `parser_name` | `str` | `""` | Parser used for extraction. |
| `raw_text` | `str` | `""` | Full extracted document text. |
| `trace` | `Trace \| None` | `None` | In-memory workflow trace. Excluded from JSON serialization. |

### What the final result gives you

When you inspect the final result, think in layers:

- `result.data` is the simple field-value payload for APIs, CSV, and downstream business logic.
- `result.fields[...]` is the rich per-field payload. Each field can carry:
  - `value` and `original_value`
  - `trust_gate`
  - `trust`
  - `ocr`
  - `consensus`
  - `verification`
  - `evidence`
  - `validation_status` and `errors`
- `result.usage` aggregates token usage and cost across extraction, retries, deciders, and reviewers.
- `result.confidence_score` is the OCR-based score for the final result. It is `None` when OCR is unavailable.
- `result.consensus_score` is the multi-instance agreement score. It is `None` when no consensus data exists.
- `result.review_status`, `result.needs_review`, `result.review_reasons`, `result.review_verdicts`,
  `result.reviewed_by`, `result.reviewed_at`, and `result.rejection_reason` describe human/LLM review state.
- `result.corrections` records human edits applied after extraction.
- `result.validation_errors` stores document-level validation problems.
- `result.escalated` and `result.escalation_reason` explain auto escalation to vision or hybrid modes.
- `result.trace_id`, `result.trace`, `result.model_name`, `result.parser_name`, and `result.raw_text`
  capture runtime and provenance metadata.

`trace` is intentionally excluded from `model_dump()` / `model_dump_json()` so serialized results stay portable.

## Result Methods

### `correct_field()`

```python
result.correct_field(
    field_name: str,
    new_value: Any,
    corrected_by: str = "",
    reason: str = "",
) -> None
```

Behavior:

- Raises `KeyError` if `field_name` does not exist.
- Stores the previous value in `field.original_value` the first time a field is corrected.
- Sets `field.value` and `result.data[field_name]` to `new_value`.
- Sets `field.corrected = True`.
- Appends a `FieldCorrection`.

### `approve()`

```python
result.approve(approved_by: str = "") -> None
```

Behavior:

- Raises `ValueError` if the result is already approved or rejected.
- Sets `review_status = "approved"`.
- Sets `reviewed_by` and `reviewed_at`.

### `reject()`

```python
result.reject(rejected_by: str = "", reason: str = "") -> None
```

Behavior:

- Raises `ValueError` if the result is already approved or rejected.
- Sets `review_status = "rejected"`.
- Sets `reviewed_by`, `reviewed_at`, and `rejection_reason`.

### `provenance()`

```python
result.provenance(field_name: str | None = None) -> dict[str, FieldProvenance]
```

Parameters:

| Parameter | Default | Description |
| --- | --- | --- |
| `field_name` | `None` | If set, returns provenance only for that field. Otherwise returns all fields. |

Returns a dict keyed by field name. Missing field names are ignored.

## Confidence And Trust Models

### `FieldTrust`

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `agreement` | `str` | `""` | Human-readable agreement such as `"4/5"`. |
| `agreement_ratio` | `float` | `0.0` | Agreement with final value. |
| `found_in_source` | `bool` | `False` | Whether the value was found in source text. |
| `valid` | `bool` | `True` | Whether validation passed. |
| `trust_gate` | `bool` | `False` | Whether the field passed the trust gate. |
| `explanation` | `str` | `""` | Human-readable trust explanation. |

### `FieldConsensus`

Present only for multi-instance extraction.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `n_instances` | `int` | `0` | Candidates requested. |
| `n_succeeded` | `int` | `0` | Candidates that returned valid JSON. |
| `agreement` | `str` | `"0/0"` | Agreement display string. |
| `agreement_ratio` | `float` | `0.0` | Fraction agreeing with final value. |
| `majority_ratio` | `float` | `0.0` | Fraction in largest candidate cluster. If greater than agreement ratio, the decider overrode the majority. |

### `OCRDocumentConfidence`

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `score` | `float` | `0.0` | Mean OCR word confidence across the document. |
| `word_count` | `int` | `0` | Number of OCR words scored. |
| `low_confidence_ratio` | `float` | `0.0` | Fraction of words below the low-confidence threshold. |

### `OCRFieldConfidence`

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `score` | `float \| None` | `None` | Minimum word confidence in the matched span. |
| `match_method` | `str` | `"unmatched"` | `"exact_block"`, `"fuzzy_block"`, `"page_text"`, or `"unmatched"`. |
| `match_ratio` | `float` | `0.0` | Exact/fuzzy match quality. |
| `matched_text` | `str` | `""` | OCR text matched to the field value. |
| `page_number` | `int \| None` | `None` | Page of the matched span. |
| `bbox` | `BoundingBox \| None` | `None` | Union bbox for single-page matches. |
| `rects` | `list[PageRect]` | `[]` | Per-line/page highlight rectangles. |

### `FieldVerification`

Created by zoom-and-verify.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `verified` | `bool` | `False` | True when the field was verified. |
| `agrees` | `bool` | `False` | True when the zoomed re-read confirmed the value. |
| `changed` | `bool` | `False` | True when a schema-valid correction was applied. |
| `original_value` | `Any` | `None` | Value before verification correction. |

## Token Usage

### `TokenUsage`

Aggregated usage across extraction candidates, deciders, JSON repair retries, LLM reviewers,
and batch/router totals.

Fields:

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `prompt_tokens` | `int` | `0` | Prompt/input tokens. |
| `completion_tokens` | `int` | `0` | Completion/output tokens. |
| `total_tokens` | `int` | `0` | Total tokens. |
| `n_llm_calls` | `int` | `0` | Number of calls contributing to the total. |
| `cost_usd` | `float \| None` | `None` | Estimated cost when LiteLLM can price the model. |

Methods:

```python
usage.merged(usage_dict: dict) -> TokenUsage
usage.combined(other: TokenUsage) -> TokenUsage
TokenUsage.from_usages(usages: list[dict]) -> TokenUsage | None
```

Use `merged()` for one call's raw usage dict. Use `combined()` for two aggregate totals.


## Common Access Patterns

```python
result.data["total"]
result.fields["total"].value
result.fields["total"].trust_gate
result.confidence_score
result.consensus_score
result.fields["total"].trust.trust_gate if result.fields["total"].trust else None
result.fields["total"].ocr.score if result.fields["total"].ocr else None
result.fields["total"].consensus.agreement if result.fields["total"].consensus else None
result.fields["total"].verification.verified if result.fields["total"].verification else None
result.fields["total"].evidence[0].text
result.fields["total"].evidence[0].bbox
result.usage.total_tokens if result.usage else None
result.review_status
result.review_reasons
result.corrections
result.raw_text[:200]
result.provenance("total")["total"]
```

Serialize:

```python
json_text = result.model_dump_json(indent=2)
payload = result.model_dump(mode="json")
```
